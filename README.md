# NeuralManifoldDynamics Datasets

This repository serves as a central collection point for data outputs and artifacts generated by the article **"NeuralManifoldDynamics: A Versioned Measurement Contract for Low-Dimensional Neural-Manifold Trajectories"** and its associated [code repository](https://github.com/NoeticDiffusion/NeuralManifoldDynamics).

## Overview

NeuralManifoldDynamics (v 2.0) is a versioned ingest-layer measurement contract for constructing and serializing low-dimensional and stratified neural-manifold proxy trajectories from EEG and fMRI feature tables. It provides a stable, reproducible, and inspectable interface between modality-specific preprocessing and downstream statistical or theoretical analysis.

Rather than claiming to be a unique data-adaptive state-discovery algorithm, it focuses on providing an auditable measurement contract that downstream analyses can compare, filter, and reinterpret.

## Key Features & Outputs

The datasets in this repository consist of self-describing HDF5 files and JSON manifests that include:

- **Canonical 3D Trajectory (`mnps_3d`)**: A low-dimensional chart with axes `[m, d, e]` aligned to metastability, deviation from optimal balance, and entropy.
- **Stratified 9D Chart (`coords_9d`)**: A finer-grained operational decomposition of the proxy families.
- **Local Dynamics & Jacobians (MNJ)**: First-order position in manifold space and second-order transformation structure, including primary 3D Jacobians and stratified 9D Jacobians.
- **Regional Dynamics**: Per-network (fMRI) and channel-group (EEG) trajectory representations, optionally including regional Jacobians.
- **Provenance & QC**: Explicit reproducibility provenance, including tracking of missing support, feature-standardization pipelines, and epoch-quality controls.

## Supported Modalities

The data outputs cover both functional Magnetic Resonance Imaging (fMRI) and Electroencephalography (EEG):
- **fMRI**: Evaluated using reference configurations like `ds000228`. Includes global trajectories and regional 3D summaries.
- **EEG/iEEG**: Evaluated using reference configurations like `ds004511` and `ds005555`. Features topology-based regional processing, optional Current Source Density (CSD) preprocessing, and regional stratified/block-Jacobian exports.

## Repository Structure

Processed runs are typically written into timestamped directories (e.g., `neuralmanifolddynamics_<dataset>_<timestamp>`). A typical dataset snapshot contains:

- `run_manifest.json` - Describes the meaning of key HDF5 paths and run parameters.
- `features_snapshot.json` - Describes the feature-layer side of the contract.
- Subject-level or run-level subdirectories including:
  - `summary.json`
  - `qc_summary.json`
  - `qc_reliability.json`
  - `.h5` files with explicit paths such as `mnps_3d`, `coords_9d/values`, `jacobian/J_hat`, `regional_mnps/`, and `extensions/tabular_exports/`.
